Architecture des styles

Matriochka de SCSS

Pour un maximum de performances, il faut le moins d’appels CSS possible (chaque appel CSS est un point de bloquage à l’affichage). Toutefois les navigateurs, si on leur dit via les media-querries quel CSS est utile pour eux, sont capables de différer le chargement de CSS inutiles au moment de l’affichage.
=> Faire 2 appels CSS, un vers default.css qui servira pour tous les cas et un vers large.css qui surcharge pour les écrans à partir de 980px. On a ainsi un seul appel bloquant et le poids du fichier appelé ne sert qu'aux mobiles.
